# ROAD SYSTEM

## 目的

RoadScene は STAND ROAD の第一印象であり、ゲーム体験の中心です。

MVP では、複雑なバトルや成長要素よりも先に「一本道を歩いている」感覚を作ります。プレイヤーが操作した結果として進行度が増え、道中で出会いが起き、カードを得ることが最優先です。

## 基本方針

- RoadScene を MVP の最初のプレイ可能体験にする
- プレイヤーは一本道を前へ進む
- 道は分岐しない
- 進行度が旅の中心指標になる
- 出会い、カード獲得、ゴール到達は進行度を基準に発生する

## MVP の画面要素

- 一本道の背景
- 旅人またはプレイヤーの位置
- 進行度
- 現在ステージ
- 取得カード数
- CollectionScene への導線
- 進む操作

## 操作

MVP では、以下のどちらかまたは両方を採用します。

- 画面上の「進む」ボタン
- キーボード入力

スマートフォンで触れることを考慮し、最初はボタン操作を必須とします。キーボード操作は補助とします。

## 画面効果・演出

- 一本道を歩いているように見える
- 背景が流れる
- 敵カードが遠くから近付いてくる

## 進行度

進行度は一本道をどれだけ進んだかを表します。

MVP では単純な数値で管理します。

```ts
type RoadProgress = {
  distance: number;
  stage: number;
  goalDistance: number;
};
```

進む操作によって `distance` が増えます。一定値に達するとカード獲得イベントやゴール判定が発生します。

## 遭遇イベント

MVP では、一定進行度ごとにスタンド使いと出会います。

出会いの結果としてカードを獲得します。最初は複雑な選択肢を持たず、出会いの文章と獲得カードの表示に集中します。

## ゴール

`distance` が `goalDistance` に到達したら旅は完了します。

ResultScene では、以下を表示します。

- 到達距離
- 取得カード数
- 出会ったスタンド使い数
- もう一度遊ぶ導線

## 将来拡張

- 道中の天候や地形
- ステージごとの雰囲気
- 選択式イベント
- AI による遭遇テキスト生成
- 道中での mock AI バトル発生

## 実装上の注意

RoadScene はゲーム状態のすべてを抱え込まないようにします。描画と入力を中心にし、進行度更新やカード獲得などのロジックは再利用できる関数または状態管理層へ分けます。
