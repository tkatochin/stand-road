# COLLECTION

## 目的

CollectionScene は、カード収集体験を支える主要機能です。

STAND ROAD のカードは単なるバトル用リソースではなく、旅の中で出会ったスタンド使いの記録です。プレイヤーは CollectionScene で、取得済みカード、未取得カード、取得日時、取得ステージを確認します。

## MVP の機能

- 取得済カード図鑑
- 未取得カードのシルエット表示
- カード詳細表示
- NEW 表示
- 取得日時
- 取得ステージ

検索やお気に入りは MVP には含めません。将来 Issue として追加します。

## 図鑑表示

CollectionScene では、全カード定義を基準にカード枠を表示します。

取得済みカードはカード名やスタンド使い名を表示します。未取得カードはシルエットまたは伏せカードとして表示します。

## 詳細表示

取得済みカードを選択すると、以下を表示します。

- スタンド使い名
- スタンド名
- 能力文章
- レアリティ
- 画像
- 取得日時
- 取得ステージ

未取得カードを選択した場合は、詳細を伏せます。MVP では「未取得」と分かる最小表示にします。

## NEW 表示

カード獲得直後は NEW 状態を付与します。

NEW 状態は CollectionScene でカード詳細を開いたタイミング、または CollectionScene に入った後の明示的な確認操作で解除します。

MVP では、解除タイミングを複雑にせず、実装しやすい方式を採用します。

## 取得履歴

カード取得時に以下を記録します。

```ts
type CardAcquisition = {
  cardId: string;
  acquiredAt: string;
  stage: number;
};
```

`acquiredAt` は ISO 8601 形式の文字列を基本とします。表示時に読みやすい形式へ変換します。

## 重複カード

MVP では、カードの重複取得は優先しません。

同じカードを再取得した場合の扱いは、最初は次のいずれかに固定します。

- すでに取得済みなら別カードを選ぶ
- 重複取得を許容するが、CollectionScene では取得済み状態だけ表示する

実装の単純さを優先し、MVP では「未取得カードを優先して獲得する」方針を推奨します。
