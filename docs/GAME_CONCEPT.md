# GAME CONCEPT

## 概要

STAND ROAD は、スタンド使いたちと出会い、カードを集めながら一本道を旅する AI ストーリーカード RPG です。

プレイヤーは一本の道を進み、道中でスタンド使いと出会います。出会いによってカードを獲得し、集めたカードの能力文章が物語やバトルの材料になります。

このゲームの核は数値の強弱ではなく、スタンド能力の文章を読み取り、状況に応じた物語として解釈することです。

## 開発コンセプト

- まず一本道を歩く体験を触れるようにする
- カードは攻撃力や防御力ではなく、能力文章を中心に設計する
- コレクションは単なる一覧ではなく、旅の記録として扱う
- AI バトルは能力文章と状況文から勝敗と理由を生成する
- MVP では本番 AI 接続より、体験の骨格を早く触れることを優先する

## コアループ

1. 一本道を進む
2. スタンド使いと出会う
3. カードを獲得する
4. コレクションでカードと旅の記録を見る
5. さらに道を進む
6. ゴールに到達し、旅の結果を見る

## MVP で重視する体験

MVP の目的は、完成されたゲームシステムを作ることではありません。最初に確認すべきことは、プレイヤーがブラウザで起動してすぐに「一本道を歩き、カードを集めている」と感じられることです。

そのため、MVP では以下を優先します。

- RoadScene で歩ける
- 進行度が変化する
- 道中でカードを獲得できる
- CollectionScene で取得済みカードと未取得カードを見られる
- ゴール到達とリザルトがある

##プレイヤーの感情

プレイヤーは未知のスタンド使いとの出会いを楽しみ、「次はどんな能力に出会えるのだろう」という期待感を持ちながら旅を続ける。


## MVP に含めないもの

- 本番 AI API 接続
- ランキング
- カード検索
- お気に入り
- デッキ構築
- 攻撃力や防御力による数値バトル

## 開発ポリシー

このプロジェクトは「人間はできるだけコードを書かない」ことを目的とした実験です。

AI は単なる実装者ではなく、自律的な開発メンバーとして振る舞います。承認された Issue の範囲内で、設計判断、実装、README 更新、ドキュメント更新、テスト追加、ビルド確認、エラー修正、リファクタリングを主体的に行います。

ただし、承認されていない Issue には着手しません。Issue ごとに完了報告を行い、人間の承認を待って次へ進みます。
